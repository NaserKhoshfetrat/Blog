Git Commands Cheat Sheet

Mastering Git: Essential Commands for Version Control Git, the powerful version control system, is an essential tool for developers. Whether you're managing a solo project or collaborating with a team, understanding key Git commands can streamline your workflow and enhance your productivity. This guide covers the most crucial Git commands every developer should know.


Basic Commands


1. git init

Description: Initializes a new Git repository in your current directory.

Example: git init myrepo

Git-scm.com doc


2. git add <file>

Description: Stages a file for the next commit.

Example: git add myfile.txt

Git-scm.com doc


3. git add .

Description: Stages all changes in the current directory and its subdirectories.

Example: git add .

Git-scm.com doc


4. git commit -m "<message>"

Description: Commits staged changes with a meaningful message.

Example: git commit -m "Added new feature"

Git-scm.com doc


5. git log

Description: Displays a log of all commits made in the repository.

Example: git log

Git-scm.com doc



Branching and Merging


1. git branch <branch-name>

Description: Creates a new branch with the specified name.

Example: git branch feature/new-feature

Git-scm.com doc


2. git checkout <branch-name>

Description: Switches to a different branch.

Example: git checkout feature/new-feature

Git-scm.com doc


3. git merge <branch-name>

Description: Merges changes from another branch into the current branch.

Example: git merge feature/new-feature

Git-scm.com doc


4. git rebase <branch-name>

Description: Rebases local changes on top of the specified branch, rewriting commit history.

Example: git rebase feature/new-feature

Git-scm.com doc


Version Control and History


1. git status

Description: Displays the status of the repository, including unstaged changes and uncommitted changes.

Example: git status

Git-scm.com doc


2. git diff

Description: Displays the differences between the current version and the previous version of a file.

Example: git diff myfile.txt

Git-scm.com doc


3. git reset <commit-hash>

Description: Resets the branch to a specific commit, discarding all changes made after that commit.

Example: git reset HEAD~1

Git-scm.com doc


4. git revert <commit-hash>

Description: Reverses the changes made in a specific commit, creating a new commit with opposite changes.

Example: git revert HEAD~1

Git-scm.com doc


Configuring Git


1. git config --global user.name "<name>"

Description: Sets the global user name for Git commits.

Example: git config --global user.name "John Doe"

Git-scm.com doc


2. git config --global user.email "<email>"

Description: Sets the global user email for Git commits.

Example: git config --global user.email "john.doe@example.com"

Git-scm.com doc


Remote Repositories


1. git remote add <name> <url>

Description: Adds a remote repository to your local repository.

Example: git remote add origin https://github.com/user/repo.git

Git-scm.com doc


2. git fetch <remote-name>

Description: Retrieves data from a remote repository, without merging changes.

Example: git fetch origin

Git-scm.com doc


3. git push <remote-name> <branch-name>

Description: Pushes local changes to a remote repository.

Example: git push origin master

Git-scm.com doc


4. git pull <remote-name> <branch-name>

Description: Fetches and merges changes from a remote repository.

Example: git pull origin master

Git-scm.com doc


Troubleshooting


1. git reset --hard

Description: Resets the branch to the last commit, discarding all local changes.

Example: git reset --hard

Git-scm.com doc


2. git clean -fd

Description: Removes untracked files and directories from the working directory.

Example: git clean -fd

Git-scm.com doc


3. git log -p <commit-hash>

Description: Displays a patch of changes made in a specific commit.

Example: git log -p HEAD~1
